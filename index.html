Topic 1: HTML5 APIs (Introduction) & SVG "HTML5 brought a lot of new capabilities to the browser, often referred to as HTML5 APIs (Application Programming Interfaces). While these APIs primarily use JavaScript to interact with them, HTML provides the elements that enable these interactions. We'll briefly touch on a few, but remember, full implementation requires JavaScript, which is for another course! Geolocation API: Allows a web page to access the user's current location (with their permission, of course). Think of map applications that show 'you are here.' Drag and Drop API: Enables elements on a webpage to be dragged and dropped, creating interactive interfaces like file uploads or reordering lists. Web Storage API (localStorage and sessionStorage): Provides a way for web applications to store data locally within the user's browser. localStorage persists data even after the browser is closed, while sessionStorage clears data when the browser tab is closed. This is useful for saving user preferences or temporary data without sending it to a server. Canvas API: This is a powerful HTML5 element () that allows you to draw graphics, animations, and even games directly on a webpage using JavaScript. It's like a blank canvas for drawing. While these APIs use JavaScript, the element itself is pure HTML. Let's look at that briefly. SVG (Scalable Vector Graphics): "Unlike the other APIs, SVG isn't a JavaScript API; it's an XML-based markup language for describing two-dimensional vector graphics. What does that mean? Vector Graphics: SVGs are defined by mathematical equations (paths, shapes, lines) rather than pixels. This means they can be scaled up or down to any size without losing quality or becoming pixelated. Think of logos, icons, or simple illustrations. Inline in HTML: You can embed SVG code directly into your HTML document!
